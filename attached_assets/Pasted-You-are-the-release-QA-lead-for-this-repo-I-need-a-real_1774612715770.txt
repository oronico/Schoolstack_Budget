Required release gates

The product is only GO if all of the following are true:

A. App health
- install succeeds
- build succeeds
- production start succeeds
- no fatal startup errors
- no deployment-specific import failures
- no obvious missing environment variable crash without graceful handling

B. Browser smoke
- landing page loads
- login page loads
- register page loads
- public underwriting flow loads
- authenticated dashboard loads after login
- no blocking console errors on core flows
- no dead buttons on primary routes

C. Core user journeys
You must test and document these specific flows:

Flow 1. Public underwriting flow
- open the public underwriting route
- step through the wizard with realistic data
- reach analysis
- reach export area
- confirm the workflow is usable
- verify no NaN, undefined, broken calculations, or dead-end states appear

Flow 2. Authenticated user flow
- register a new user or use a valid test user
- log in
- open dashboard
- create a model or open an existing one
- save it
- reload it
- edit it
- verify changes persist

Flow 3. Export flow
From a real model, export all of these:
- Formula Export XLSX
- Lender Pro Forma XLSX
- Underwriting V2 XLSX
- at least one PDF report
- if more PDF exports are visible in product, test them too

D. Export validation
For each XLSX export:
- confirm HTTP response is correct
- confirm content-type is XLSX
- confirm content-disposition is sensible
- verify file starts with ZIP magic bytes PK
- verify workbook can actually be opened by a library
- verify expected sheet names exist
- verify workbook is not corrupt
- save one sample into /qa/artifacts/exports/

For each PDF export:
- confirm HTTP response is correct
- confirm content-type is PDF
- verify file begins with %PDF
- verify file is not blank
- verify PDF opens successfully
- save one sample into /qa/artifacts/exports/

E. Proxy and binary safety
This is critical.

Test binary downloads through the same route external users will use.

Do not only test internal generation helpers.
Do not only test local file creation.
Test the actual HTTP download behavior.

Specifically verify:
- downloads are not corrupted
- HTML error pages are not being returned instead of files
- Netlify or proxy layers are not mangling binary responses
- file size is reasonable
- workbook opens cleanly

Specific things to watch for

1. Export corruption
This repo has had recent work around binary export handling. Validate the actual downloaded files carefully.

2. Sheet integrity
Do not just check that a file exists. Open the workbook programmatically and inspect sheet names.

3. Calculation sanity
Flag any undefined, NaN, broken totals, missing DSCR values, or unusable summary outputs.

4. Auth behavior
Protected routes should behave correctly with and without authentication.
