**Diagnosis**
- The TypeScript build stops at [line 67](#L67)–[line 76](#L76) because functions like `createInsertSchema`/`createSelectSchema` are being called with a full `z.object({...})` as the override argument. In current `drizzle-zod` typings, that parameter must be a plain object whose values are the override schemas, not a `ZodObject`. Passing the wrong type now violates the generic constraint (`TS2344`).

**Solution**
- In each schema file under `lib/db/src/schema/`, replace the `z.object({...})` override argument with an object literal where each property is the `Zod` schema you want to override. For example:

  ```ts
  const metadataSchema = z
    .record(z.string(), z.unknown())
    .nullable()
    .optional();

  export const insertEventSchema = createInsertSchema(events, {
    metadata: metadataSchema,
  });

  export const selectEventSchema = createSelectSchema(events, {
    metadata: metadataSchema,
  });
  ```

- Apply the same pattern to `exports.ts`, `financial-models.ts`, `schools.ts`, and `users.ts`, mapping each column to its override schema rather than wrapping the overrides in `z.object`. After updating these calls, rerun the build—the type check should pass.
