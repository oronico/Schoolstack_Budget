Feature 3: Days Cash on Hand (DCOH) Everywhere
Why: DCOH is the #1 liquidity metric every CDFI, charter authorizer, and Bellwether reviewer checks first. The formula is simple: (Ending Cash / (Total Annual Expenses / 365)). SchoolStack already computes it in the DSCR & Covenants tab of the 21-tab underwriting workbook (line 2164), and the covenantThresholds.minDaysCashOnHand field exists — but it's absent from the Financial Health dashboard, the consultant engine output, and the lender pro forma.
Current state:

Computed in underwriting-workbook.ts line 2164–2167 (PASS/FAIL check only)
covenantThresholds.minDaysCashOnHand schema field exists (default 45)
NOT in: Financial Health dashboard, consultant engine, lender pro forma, standard export

Changes Required

Add helper function to workbook-helpers.ts:

typescriptexport function computeDaysCashOnHand(endingCash: number, totalAnnualExpenses: number): number {
  if (totalAnnualExpenses <= 0) return endingCash >= 0 ? 365 : 0;
  return Math.max(0, (endingCash / totalAnnualExpenses) * 365);
}

Add benchmark thresholds to benchmark-thresholds.ts:

// Below 45 = red
