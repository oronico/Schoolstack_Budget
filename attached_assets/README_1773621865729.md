# SchoolStack Brand System

## Overview

This brand system provides the visual identity, design tokens, and asset library for the SchoolStack product suite. It is designed to feel warm, professional, and trustworthy — like expert guidance delivered with kindness.

SchoolStack is powered by Building Hope Impact Fund.

## How to Apply

### Typography
- **Headlines & Display**: Use Quicksand Bold
- **Body & UI text**: Use Nunito Regular
- **Buttons, labels, subheads**: Use Nunito SemiBold or Bold
- Load both fonts from Google Fonts:
  ```html
  <link href="https://fonts.googleapis.com/css2?family=Quicksand:wght@500;600;700&family=Nunito:wght@400;500;600;700&display=swap" rel="stylesheet">
  ```

### Colors
- **Deep Navy (#1E293B)** — structural work: headings, body text, navigation
- **Cream (#FAF9F7)** — dominant background color
- **Amber (#D97706)** — signature accent for buttons, highlights, brand moments. Never use as paragraph text.
- **Teal (#0D9488)** — action color for CTAs, next steps, and guidance
- **Sage Green (#16A34A)** — success states only
- **Rose (#E11D48)** — alerts and errors only

### Extension Colors
- **Space Blue (#5B7CFA)** — SchoolStack Space product accent, data visualizations, charts
- **Evergreen (#328555)** — SchoolStack Budget product accent, financial contexts, depth

### Product-Specific Accents
- **SchoolStack** (flagship) → Amber + Teal
- **SchoolStack Space** → Amber + Space Blue (#5B7CFA). Space Blue renders on both light and dark backgrounds.
- **SchoolStack Budget** → Amber + Evergreen (#328555). Evergreen renders on both light and dark backgrounds.

### Design Tokens
Import `schoolstack-tokens.json` for consistent values across all products. The token file includes:
- Color palette (hex values)
- Typography (fonts, weights, sizes)
- Spacing scale
- Border radius scale
- Shadow definitions
- Product accent mappings

### Logo Usage
- Always maintain clear space around the logo (minimum: height of the mark)
- Use the horizontal lockup as the primary logo
- Use the stacked lockup when horizontal space is limited
- Use the icon-only mark for favicons, app icons, and small contexts
- Never add effects, gradients, or outlines to the logo
- SVG logos use `font-family: Quicksand` — ensure the font is loaded or use the outlined variants for contexts where the font may not be available

### Brand Architecture
All products share:
- Same wordmark logic (Quicksand Bold)
- Same icon geometry (three rounded bars)
- Same spacing system
- Same typography
- Same rounded-corner language (0.625rem default radius)
- Same base palette (Cream, Deep Navy, Amber)

Products are distinguished by their secondary accent color and the colors used in their three-bar mark.

### Voice Guidelines
- Guide, don't instruct — use "let's" and "we"
- Normalize learning — treat every question as valid
- Be specific, be kind — tell users what's needed without blame
- Frame complexity as something to work through, never a failure

### Notes on exported SVGs
- Logo SVGs embed Quicksand Bold (Latin subset, ~15KB woff2) as a base64 `@font-face` in `<defs>`. They render correctly in any modern browser, design tool, or OS preview without requiring the font to be installed.
- Mark-only SVGs (favicons, icon marks) contain no text and are fully self-contained at <1KB each.
- For print workflows or contexts where embedded web fonts are not processed, convert text-based SVGs to outlined paths using a vector editor (Figma: right-click → Outline stroke; Illustrator: Type → Create Outlines).

## Final Test

The brand should:
- Feel like one family across all products
- Make each product distinct at a glance
- Feel warmer than typical finance or ops software
- Look credible to funders, borrowers, school leaders, and partners
- Convey: "You are in capable hands, and this is going to be easier than you thought."
