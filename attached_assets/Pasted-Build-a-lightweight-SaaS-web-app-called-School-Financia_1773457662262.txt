Build a lightweight SaaS web app called “School Financial Model Builder by SchoolStack.ai.”

PRODUCT GOAL
This is a teaser product that launches before the full SchoolStack.ai platform.
Its purpose is to help school founders create a simple, lender-friendly 5-year financial model, save their work, revisit it later, get practical CFO-style guidance, and export the model to Excel.

This should feel polished, trustworthy, and calm.
Simple, but not ugly.
It should not try to be the full SchoolStack platform.

PRIMARY USER
A school founder, microschool founder, or small private/charter school operator with limited finance expertise.

CORE USER STORY
A founder creates an account, starts a school model, enters assumptions through a clean guided flow, saves progress automatically, receives practical CFO-style feedback, and exports a clean .xlsx workbook with assumptions and formulas.

PRODUCT POSITIONING
This is not just an Excel exporter.
This is a lightweight “fractional CFO for school founders” experience.

The consultant experience should feel like:
- practical
- plain-English
- supportive but candid
- lender-aware
- specific about risks and tradeoffs

Do NOT make it feel like a generic chatbot toy.

V1 PRODUCT SCOPE
Build the following:

1. User authentication
2. Saved user dashboard
3. Create, edit, duplicate, archive, and delete saved financial models
4. Guided multi-step model builder
5. CFO consultant insights panel
6. Excel export with assumptions page and formulas

DO NOT BUILD

CFO CONSULTANT REQUIREMENTS
This is a core feature.

Important:
Do NOT make v1 a free-form chat product.
Instead, make it a structured consultant experience that feels like a smart fractional CFO reviewing the model.

After assumptions are saved and outputs are calculated, generate a consultant panel with:

1. Executive summary
2. Biggest strength
3. Biggest risk
4. Three recommended actions
5. Lender readiness status
6. Key metrics with interpretation

Possible lender readiness statuses:
- Strong
- Needs work
- Not yet ready

The consultant should speak in plain English and explain what the numbers mean.

Examples of consultant observations:
- payroll is too high relative to revenue
- rent burden is heavy
- enrollment growth assumptions are aggressive
- founder salary may be unrealistically low
- margins improve by year 3
- revenue per student is strong
- staffing model looks sustainable

CONSULTANT ENGINE
Build this as a deterministic rules engine first.
Do not depend on an external LLM for the core feature set.

The consultant engine should:
- read the calculated model outputs
- evaluate simple thresholds and patterns
- generate structured narrative guidance
- store the consultant output in the database

Architect the code so an AI narrative layer could be added later, but keep v1 robust without needing an API key.

MODEL LOGIC
Keep the logic simple and transparent.

Enrollment:
- use annual student counts from assumptions

Revenue:
- tuition revenue = students × tuition per student
- ESA revenue = students × ESA revenue per student
- other earned revenue = students × other revenue per student
- gross revenue = sum
- scholarship discount = applicable revenue × discount rate
- net revenue = gross revenue - discount + grants/fundraising

Staffing:
- teacher count = round up(students / students per teacher)
- teacher payroll = teacher count × teacher salary
- admin payroll = admin count × admin salary
- founder salary = founder salary input
- benefits = total salary base × benefits rate
- total staffing cost = sum

Operating expenses:
- annual rent = monthly rent × 12, escalated annually
- utilities = annual utilities
- insurance = annual insurance
- curriculum = students × curriculum per student
- technology = students × technology per student
- marketing = annual marketing
- other operating expenses = annual other operating expenses
